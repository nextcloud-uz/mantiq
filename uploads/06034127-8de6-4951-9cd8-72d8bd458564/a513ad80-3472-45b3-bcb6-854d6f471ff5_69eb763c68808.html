Поставка высокопроизводительного GPU-сервера для обучения моделей ИИ и оказание услуг по технической подготовке специалистов

1. Обзор проекта

Целью данного лота является создание высокопроизводительной вычислительной инфраструктуры для локальной разработки, дообучения и развертывания мультимодальных моделей ИИ. Проект включает поставку специализированного GPUсервера, развертывание масштабируемой MLOps-среды и комплексную техническую подготовку штатных специалистов для самостоятельного запуска ИИ-сервисов.

3. Технические требования (Technical Requirements)

3.1. Сетевая инфраструктура и управление данными

  •  Локальные API и сетевое взаимодействие: Настройка защищенного внутреннего контура (Local Network) для бесперебойного обмена данными между микросервисами (API для машинного перевода, Speech-to-Text и Image-to-Text).
  •  Синхронизация датасетов: Интеграция с локальными или облачными хранилищами (Nextcloud) через протоколы WebDAV и утилиту Rclone для автоматизированной выгрузки и синхронизации объемных аудиои текстовых датасетов с Hugging Face или приватных репозиториев.
  •  Безопасный удаленный доступ: Организация VPN-туннелей (WireGuard/IPsec) для безопасного удаленного администрирования сервера и прямого доступа к средам разработки.

3.2. Виртуализация, контейнеризация и оркестрация

  •  Гипервизор и управление ресурсами: Установка и базовая настройка Proxmox VE. Конфигурирование проброса видеокарты (PCIe GPU Passthrough для RTX 6000 Ada) в выделенные виртуальные машины для минимизации потери производительности.
  •  Автоматизация рабочих процессов: Развертывание VM в связке с stateful-базами данных для автоматизации рутинных задач, обработки промптов и генерации объемной технической документации.
  •  Оркестрация микросервисов: Развертывание Docker-среды с NVIDIA Container Toolkit. Настройка кластера Kubernetes для управления изолированными контейнерами веб-платформы и обеспечения их отказоустойчивости.

3.3. Настройка среды вычислений (MLOps)

  •  Серверная ОС: Установка и тюнинг Ubuntu 24.04 LTS (ядро, параметры памяти).
  •  Мониторинг: Настройка системы оповещения о доступности камер и статусе работы графических процессоров. 
  •  AI Стек: Развертывание NVIDIA DeepStream SDK 6.x для дешифрации и обработки потоков. Настройка Triton Inference Server для управления моделями распознавания лиц в реальном времени.

3.4. Требования к оказанию услуг по обучению штатных специалистов

В рамках проекта разрабатывается и проводится комплексная программа практического обучения для штатных сотрудников Заказчика. Обучение проводится с использованием поставляемого оборудования (GPU-сервер) и строго ориентируется на стек технологий, используемый в проектах Заказчика.

Программа обучения включает следующие обязательные модули:

Модуль 1. Практикум для инженеров машинного обучения (AI/ML)

Для команды проводится обучение по следующим направлениям:

  •  Тонкая настройка (Fine-tuning) моделей: Проводятся практические занятия по дообучению открытых LLM и ASR архитектур (Whisper, Wav2Vec2, Gemma) с применением методов оптимизации памяти (PEFT, LoRA/QLoRA) на поставляемых видеокартах.
  •  Развертывание (Inference): Проводится подготовка специалистов к упаковке и деплою дообученных моделей в production-среду с использованием vLLM, Text Generation Inference (TGI) или Triton Inference Server для обеспечения высокой пропускной способности API.
  •  Работа с малоресурсными языками: Осуществляется обучение методикам подготовки, аугментации и версионирования (DVC) обучающих датасетов, специфичных для узбекского и таджикского языков.

Модуль 2. Инфраструктура, DevOps и MLOps

Для системных администраторов Заказчика организуется обучение администрированию высоконагруженных GPU-кластеров:

  •  Виртуализация и оркестрация: Отрабатывается настройка и управление гипервизором Proxmox VE, включая практическую реализацию прямого проброса видеокарт (PCIe GPU Passthrough) в виртуальные машины. Изучается оркестрация микросервисов через Kubernetes и Docker с использованием NVIDIA Container Toolkit.
  •  Управление хранилищами данных: Производится настройка частных облачных хранилищ (Nextcloud/TrueNAS) и проводится обучение автоматизации

синхронизации терабайтных датасетов с внешними репозиториями (Hugging Face) посредством Rclone и WebDAV.

  •  Мониторинг кластера (Observability): Осуществляется практика развертывания стека Prometheus + Grafana для мониторинга аппаратной части сервера (интеграция метрик NVIDIA-SMI: VRAM, температура, энергопотребление GPU) и состояния NVMe-накопителей.

Модуль 3. Разработка бэкенда и автоматизация рабочих процессов

Программа для backend-разработчиков охватывает:

  •  Интеграция ИИ-сервисов: Разбираются архитектурные паттерны построения отказоустойчивых API-шлюзов для объединения локальных нейросетевых микросервисов (машинный перевод, Speech-to-Text, Image-to-Text) в единую экосистему.
  •  Платформы автоматизации (n8n): Обеспечивается глубокое погружение в среду n8n. Проводится обучение созданию сложных логических пайплайнов (workflows) в связке с базами данных (stateful) для автоматической генерации технической документации и обработки потоковых промптов.

Модуль 4. Фронтенд-разработка и UI/UX интеграция

Обучение клиентской части включает:

  •  Интеграция Streaming API: Демонстрируются практические примеры подключения пользовательских интерфейсов к потоковым ИИ-сервисам (например, для плавного вывода генерируемого LLM текста или транскрипции в реальном времени).
  •  Современные стандарты UI: Осваивается реализация отказоустойчивых пользовательских интерфейсов для взаимодействия с высоконагруженными нейросетевыми бэкендами.

Формат и результаты обучения:

По итогам проведения курсов Заказчику предоставляются видеозаписи всех практических сессий, а также методические материалы и скрипты развертывания, адаптированные под инфраструктуру.
